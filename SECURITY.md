# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in OSINT Forge, **do not open a public issue**.

Please report it responsibly by sending an email to:

**security@osint-forge.com**

Include in your report:
- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Any relevant logs, screenshots, or proof-of-concept

We will acknowledge your report within **48 hours** and aim to provide a fix or mitigation plan within **7 days**, depending on severity.

We appreciate responsible disclosure and will credit researchers who report valid vulnerabilities (unless anonymity is requested).

---

## Reporting Bugs & Other Issues

For any other bug, error, or non-security issue, please open an issue directly in the repository:

**https://github.com/OSINT-Forge-Community/osint-forge-docs/issues**

When creating an issue, use the appropriate label(s) to help us triage it faster:

| Label | Description |
|---|---|
| `flag-validation` | Incorrect flag validation or challenge verification logic |
| `challenge-issue` | Quest or challenge problem (instructions, data, validation) |
| `ctf-issue` | OSINT CTF event issue (registration, visibility, results) |
| `ranking-issue` | Leaderboard, ranking, scoring, or XP calculation problem |
| `profile-issue` | User profile issue (XP, rank, badges, stats, settings) |
| `misconfiguration` | Configuration/secrets/env/infra misconfiguration |
| `dependency-issue` | Dependency vulnerability/upgrade conflict/breaking change |
| `regression` | Previously working behavior broken after a change |
| `performance` | Performance, latency, throughput, scalability |
| `runtime-error` | Crashes, exceptions, 5xx, runtime failures |
| `critical-bug` | Blocking bug / major impact on users or production |
| `bug` | Functional bug / unexpected behavior |
| `security-report` | External report / responsible disclosure tracking |
| `vulnerability` | Confirmed or likely vulnerability (auth, injection, access control, etc.) |
| `security` | Security-related issue requiring attention |
