# Contributing to OSINT Forge

Thank you for your interest in OSINT Forge.
This document explains how you can participate in this project.

> **Important:** This repository contains **documentation only**.
> The OSINT Forge platform source code is **private** and is not hosted here.
> We do not accept code contributions, pull requests on the platform codebase,
> or any external involvement in the application development.

---

## Before Participating

Please read and agree to:

- [Code of Conduct](./CODE_OF_CONDUCT.md)
- [Security Policy](./SECURITY.md)
- [Disclaimer](./DISCLAIMER.md)

---

## What You Can Do

| Type | Where | How |
|---|---|---|
| Report a platform bug | [GitHub Issues](https://github.com/OSINT-Forge-Community/osint-forge-docs/issues) | Use the appropriate label |
| Report a challenge issue | [GitHub Issues](https://github.com/OSINT-Forge-Community/osint-forge-docs/issues) | Use `challenge-issue` label |
| Report a security vulnerability | Email only | See [SECURITY.md](./SECURITY.md) |
| Share feedback or suggestions | Discord or Issues | Open a discussion |

---

## Reporting Issues

1. **Search first** â€” check that the issue does not already exist
2. **Use a clear title** â€” describe the problem concisely
3. **Add the right label** â€” see the label list below
4. **Provide context** â€” steps to reproduce, expected vs actual behavior,
   screenshots if relevant

### Available Labels

| Label | Use for |
|---|---|
| `bug` | Functional bug / unexpected behavior |
| `critical-bug` | Blocking bug / major impact on users |
| `runtime-error` | Crashes, exceptions, 5xx errors |
| `regression` | Previously working behavior now broken |
| `performance` | Performance, latency, scalability issues |
| `challenge-issue` | Quest or challenge problems |
| `ctf-issue` | OSINT CTF event issues |
| `flag-validation` | Flag or challenge verification logic |
| `ranking-issue` | Leaderboard, scoring, XP problems |
| `profile-issue` | User profile, badges, settings |
| `misconfiguration` | Config, env, infrastructure issues |
| `dependency-issue` | Dependency conflicts or vulnerabilities |
| `security` | Security-related concern |
| `vulnerability` | Confirmed or suspected vulnerability |
| `security-report` | External responsible disclosure |

> **Security issues:** Never open a public issue for vulnerabilities.
> Email [security@osint-forge.com](mailto:security@osint-forge.com) instead.
